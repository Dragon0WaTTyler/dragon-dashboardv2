## Remaining authorization boundary

Dragon is runnable and feature-complete for local normalized data. It is not yet
populated with the user's personal records. There is intentionally no migration
apply command: implementing/importing/reconciling personal records requires a
reviewed domain mapping report, a target backup, and explicit approval. External
provider sync also requires credentials and adapter configuration outside Git.
