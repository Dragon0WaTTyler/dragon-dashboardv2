# M1 delivery record

Status: complete on 2026-07-14.

## Scope delivered

- Standalone Git repository and Python 3.13 project with pinned dependency lock,
  Ruff, pytest, coverage, and GitHub Actions CI.
- Flask application factory with isolated configuration overrides, Blueprints,
  unbound SQLAlchemy, Alembic, Flask-Login, and Flask-WTF/CSRF extensions.
- Typed environment settings, fail-fast production secret validation, safe
  feature flags, ignored instance storage, JSON logging, request IDs, security
  headers, and unified HTML/API error handling.
- Initial Alembic-managed local-user schema plus interactive administrator
  creation and password-rotation commands. No default credentials exist; only a
  Werkzeug scrypt password hash is persisted.
- Deployment and API health routes, protected Today shell, login, CSRF-protected
  logout, session renewal, and a protected design-system page built from the
  same Jinja macros used by production templates.
- A new light-first editorial/productivity interface using local IBM Plex assets,
  CSS tokens, separated layout/component styles, and vanilla JavaScript. The
  responsive shell includes desktop module navigation and mobile bottom
  navigation; future modules are explicit disabled placeholders.

## Verification evidence

- Runtime: Python 3.13.14 and Flask 3.1.3.
- Tests: 25 passed with 93% application statement coverage.
- Static analysis: Ruff clean.
- Database: empty-database upgrade reached Alembic revision `b86bbec3d904` and
  created only `alembic_version` and `users` tables.
- Browser smoke: passed on desktop and 390 px mobile, including keyboard focus,
  dialog Escape/focus restoration, semantic landmarks, one H1, form labels,
  minimum touch targets, and horizontal-overflow checks.
- Frontend policy scan: zero inline styles, zero inline scripts, zero
  `!important` declarations, and zero external assets.
- Network denial: every M1 GET route completed while outbound HTTP/socket calls
  were blocked by tests.
- Repository safety: ignored runtime database and local secret were removed;
  the tracked-file secret scan is a required final and CI gate.

## Deferred by scope

M2 and later own domain persistence, snapshots, migration inventory/apply,
feature pages, external integrations, API tokens, dark mode, and release
hardening. The legacy project remains a read-only reference and was not modified.
